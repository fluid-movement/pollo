# PO File Concepts and Internal Data Model

## PO File Constructs

| Construct | Description |
|---|---|
| `msgid ""` (header) | First entry with empty msgid; contains file metadata. Never translated. |
| `msgid` / `msgstr` | Standard singular translation unit |
| `msgid_plural` / `msgstr[n]` | Plural unit; form count from `Plural-Forms` header |
| `msgctxt` | Context disambiguator; part of the unique entry key |
| `# ` (space after `#`) | Translator comment — written by humans |
| `#.` | Extracted comment — generated by xgettext from source |
| `#:` | Source reference `file:line` — preserved but never sent to LLM |
| `#,` | Flags — e.g. `fuzzy`, `c-format`, `python-format` |
| `#\|` | Previous source strings (`#\| msgid`, `#\| msgid_plural`) on fuzzy entries |
| `#~` | Obsolete entry — prefixes every line of the block |

## Entry States

| State | Condition |
|---|---|
| `untranslated` | Singular: `msgstr` is empty or whitespace-only. Plural: any `msgstr[n]` is empty or whitespace-only. |
| `fuzzy` | Entry has `fuzzy` in its `#,` flags, regardless of msgstr content. |
| `translated` | No fuzzy flag; all msgstr values non-empty and non-whitespace. |

A fuzzy entry with a non-empty msgstr is still `fuzzy`, not `translated`. The
agent confirms it by calling `set`.

## Counting Rules

`total`, `translated`, `fuzzy`, and `untranslated` counts always exclude:
- The header entry (`msgid ""`)
- Obsolete entries (`#~`)

## Internal Data Model

The following describes the logical structure the implementation must represent.

### File

A parsed PO file consists of:

- An ordered list of **nodes**, each of which is either an active entry or an
  obsolete block. This ordering must be preserved on write to produce clean
  diffs.
- A reference to the **header entry** (the entry with empty msgid); `nil`/absent
  if the file has none.
- The resolved **plural count** (`nplurals`) for the target language.
- The **language tag** (BCP 47, e.g. `"de-AT"`) from the header.
- The **language display name** (e.g. `"German (Austria)"`).

### Entry

Each active entry carries:

| Field | Source | Notes |
|---|---|---|
| Translator comment | `# ` lines | Joined; prefix stripped |
| Extracted comment | `#.` lines | Joined; prefix stripped |
| References | `#:` lines | Parsed and preserved; never sent to LLM |
| Flags | `#,` line | Split by comma and trimmed |
| Previous msgid | `#\| msgid` | Multi-line capable; unescaped |
| Previous msgid_plural | `#\| msgid_plural` | Multi-line capable; unescaped |
| msgctxt | `msgctxt` keyword | Unescaped; empty if absent |
| msgid | `msgid` keyword | Unescaped; multi-line joined |
| msgid_plural | `msgid_plural` keyword | Unescaped; empty if singular |
| msgstr | `msgstr` keyword | Singular translation; empty if untranslated |
| msgstr[n] | `msgstr[n]` keywords | Plural translations by index; absent if singular |

### Entry Key

An entry's unique key is formed as:
- If `msgctxt` is present: `<msgctxt> + U+0004 + <msgid>`
- Otherwise: `<msgid>`

U+0004 (EOT) is the gettext convention for separating context from source string.

### Obsolete Nodes

Obsolete (`#~`) blocks are stored as raw unparsed line sequences at their
original position in the node list. They are never decoded and are written back
verbatim.

## Edge Cases and Constraints

| Case | Behaviour |
|---|---|
| `.pot` file | Treated identically to `.po` |
| Header entry | Never returned by `get`, never writable via `set` |
| Obsolete `#~` entries | Preserved at original node list position; never counted or surfaced |
| Whitespace-only `msgstr` | Treated as untranslated |
| Partial plural (some forms empty) | State is untranslated; agent must provide all forms |
| Duplicate msgid | Both retained; `get`/`set` target first match; parse warning emitted |
| Non-UTF-8 charset header | Fatal error |
| Absent charset header | Accepted; treated as UTF-8 |
| Invalid UTF-8 in flag value | Structured error, exit 1 |
| `Plural-Forms` absent, language known | CLDR lookup; parse warning |
| `Plural-Forms` absent, language unknown, plural entries | Fatal error |
| `Plural-Forms` absent, language unknown, no plural entries | nplurals=1, no error |
| `Language` header absent | `""` in JSON output; `language_name` is `""` |
| `--skip-ids-file` path missing | Treated as empty skip list; no error |
| Key in skip file not in PO file | Silently ignored |
| Non-contiguous `msgstr[n]` | Fatal parse error |
| `#~` entries with `msgid_plural` | Treated as raw lines; not decoded |
| Windows `\r\n` line endings | Normalised to `\n` on parse; writer always emits `\n` |
| Empty file | Zero entries; `stats` returns all-zero counts |
| No header in file | `Language` and `Plural-Forms` unresolvable from header |
| Concurrent writes | Out of scope; tool assumes exclusive file access |
